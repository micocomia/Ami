## 1. Context

The AI Tutor system infers learner skill levels from resume/background text using an LLM. Currently there are **zero checks** for whether these inferences are biased by demographic cues (name, gender, university prestige, nationality, etc.). This module adds a post-processing bias audit layer that reviews skill gap assessments for fairness â€” completely non-overlapping with existing functionality.

---

## 2. Objectives

1. **Bias flag detection** (LLM) â€” Scans each skill gap's `reason` for assumption-based rather than evidence-based reasoning
2. **Demographic-blind validation** (LLM) â€” Checks if assessments are influenced by name, gender, age, institution prestige, or nationality
3. **Confidence calibration check** (deterministic) â€” Flags low-confidence + extreme-level combinations (e.g., `confidence=low` + `level=unlearned`)
4. **Ethical disclaimer** â€” Adds transparency metadata informing learners assessments are AI-generated
5. **Frontend display** â€” Makes all of the above visible to the user in the Skill Gap page

---

## 3. Architecture

Strictly a **post-processing layer** â€” takes existing skill gap results as input, returns an audit report. Zero modifications to the existing pipeline.

```
POST /identify-skill-gap-with-info  (existing, unchanged)
  -> returns skill_gaps

POST /audit-skill-gap-bias  (NEW)
  -> takes skill_gaps + learner_information
  -> returns bias audit report
```

## 5. Implementation Steps

### Step 1: Schemas (`backend/modules/skill_gap/schemas.py`)

Add at the end of the file:

- **`BiasCategory`** enum (str, Enum): `demographic_inference`, `prestige_bias`, `gender_assumption`, `age_assumption`, `nationality_assumption`, `stereotype_based`, `unsubstantiated_claim`
- **`BiasSeverity`** enum (str, Enum): `low`, `medium`, `high`
- **`BiasFlag`** model:
  - `skill_name: str` â€” Name of the skill gap being flagged
  - `bias_category: BiasCategory` â€” Category of the potential bias detected
  - `severity: BiasSeverity` â€” Severity of the potential bias
  - `explanation: str` â€” Brief explanation of why this was flagged (max 40 words, validated)
  - `suggestion: str` â€” Suggested remediation (max 30 words, validated)
- **`ConfidenceCalibrationFlag`** model:
  - `skill_name: str`
  - `current_level: LevelCurrent`
  - `level_confidence: Confidence`
  - `issue: str` â€” Description of the calibration issue
- **`BiasAuditResult`** model:
  - `bias_flags: List[BiasFlag]` â€” LLM-detected bias flags
  - `confidence_calibration_flags: List[ConfidenceCalibrationFlag]` â€” Deterministic flags
  - `overall_bias_risk: BiasSeverity` â€” Overall bias risk (low/medium/high)
  - `audited_skill_count: int` â€” Number of skill gaps audited
  - `flagged_skill_count: int` â€” Number of skill gaps with at least one flag
  - `ethical_disclaimer: str` â€” Default text: "This skill gap assessment was generated by an AI system. It infers proficiency levels from the information you provided and may not fully reflect your actual abilities. If you believe any assessment is inaccurate, please adjust it manually or provide additional information about your experience."

### Step 2: Prompts (`backend/modules/skill_gap/prompts/bias_auditor.py`)

- **System prompt**: Defines the Bias Auditor role with directives to:
  - Scan for assumption-based reasoning vs evidence-based
  - Detect demographic-adjacent influence (name, gender, age, prestige, nationality)
  - Use the 7 bias categories for classification
  - Be calibrated, not alarmist (empty flags list is valid)
  - Assign `overall_bias_risk` (low/medium/high) based on flag count and severity
  - NOT re-assess skill levels â€” only audit the reasoning fairness
  - Output valid JSON matching the defined format
- **Task prompt**: Takes `{learner_information}` and `{skill_gaps}` as placeholders
- Update `prompts/__init__.py` to export the new prompts

### Step 3: Agent (`backend/modules/skill_gap/agents/bias_auditor.py`)

- **`BiasAuditPayload`**: Pydantic input model with `learner_information` (str) and `skill_gaps` (dict)
- **`BiasAuditor(BaseAgent)`**:
  - Constructor: `super().__init__(model, system_prompt, jsonalize_output=True)` â€” no tools needed
  - `audit_skill_gaps()` method:
    1. Validate input with `BiasAuditPayload`
    2. Serialize `skill_gaps` to JSON string for the prompt
    3. Call `self.invoke()` â€” LLM returns `bias_flags` + `overall_bias_risk`
    4. Run deterministic `_check_confidence_calibration()` â€” flags low-confidence + extreme-level combos
    5. Merge LLM output with deterministic flags
    6. Compute `audited_skill_count` and `flagged_skill_count`
    7. Promote `overall_bias_risk` to "medium" if calibration flags exist but LLM said "low"
    8. Validate with `BiasAuditResult` Pydantic model, return `model_dump()`
  - `_check_confidence_calibration()` static method: flags cases where `level_confidence` is "low" AND `current_level` is "unlearned" or "expert"
- **`audit_skill_gap_bias_with_llm()`**: Top-level convenience function
- Update `agents/__init__.py` and `__init__.py` to export

### Step 4: API Layer

- **`api_schemas.py`**: Add `BiasAuditRequest(BaseRequest)` with `learner_information: str` and `skill_gaps: str`
- **`main.py`**: Add endpoint after the existing `/identify-skill-gap-with-info` block (~line 503):
  - Parse `skill_gaps` string with `ast.literal_eval` (matching existing pattern)
  - Call `audit_skill_gap_bias_with_llm(llm, learner_information, skill_gaps)`
  - Return result, wrap in try/except

## 8. Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| Post-processing layer, not inline | Zero risk of breaking existing pipeline; can be toggled independently |
| Placed inside `modules/skill_gap/` not a new top-level module | Consistent with existing project structure |
| LLM-based + deterministic checks combined | LLM handles nuanced bias detection; deterministic checks catch objective inconsistencies |
| "Be calibrated, not alarmist" prompt directive | Prevents over-flagging which would erode user trust |
| Ethical disclaimer always shown | Transparency is valuable regardless of whether bias is detected |
| Max 40-word explanation, 30-word suggestion | Follows existing pattern of constrained LLM output (existing `reason` is max 20 words) |
