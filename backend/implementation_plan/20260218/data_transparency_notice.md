# Data Transparency & Privacy Notice — Implementation Plan

**Branch:** `sprint-3-ethics-bias-enhancement`
**Date:** 2026-02-18
**Project:** GenMentor (5902Group5)

---

## 1. Context

GenMentor collects learner data (resumes, learning goals, behavioral metrics) and sends it to external LLM APIs to generate skill assessments, learner profiles, and learning content. The existing Bias Auditor and Fairness Validator modules already provide per-feature ethical disclaimers, but there is no centralised data transparency notice informing users about overall data collection, AI usage, and their rights. This plan adds a team-facing data practices document and an in-app transparency notice — both purely informational with zero impact on user flow.

---

## 2. Objectives

1. **Data practices document** — A markdown file for team members and reviewers documenting what data is collected, how it is stored, what is sent externally, retention policies, and deletion behaviour
2. **In-app transparency notice** — A collapsible expander on the Onboarding page summarising data collection, AI usage, and user rights

---

## 5. Implementation Details

### Step 1: Data Practices Document (`docs/data_practices.md`)

Covers 8 sections:

1. **Data We Collect** — Table of 10 data types (credentials, persona, goal, resume, skill gaps, profile, quiz scores, session timing, events, chat messages) with collection point and whether required
2. **How Data Is Stored** — Local JSON files under `backend/data/users/` (credentials, profiles, user state, events); ChromaDB for course material embeddings; bcrypt-hashed passwords
3. **Data Sent to External Services** — LLM API (what data each feature sends), web search providers (DuckDuckGo default), embedding models (local Sentence Transformers)
4. **Data Retention** — All user data kept until account deletion; events capped at 200 per user; course embeddings permanent (not user-specific)
5. **Account Deletion** — Removes credentials, profiles, events, and user state; username becomes re-registrable
6. **AI Transparency** — Lists all 5 disclosure points already in the app (skill gap disclaimer, profile disclaimer, bias audit banners, fairness banners, data transparency notice)
7. **Authentication & Session Security** — Bcrypt passwords, JWT with 24h expiry, token-based (no cookies)
8. **What GenMentor Does NOT Do** — No data selling, no advertising, no post-deletion retention, no cross-site tracking, no real-name requirement

### Step 2: In-App Notice (`frontend/pages/onboarding.py`)

Add after the "Begin Learning" button section, still inside the `center` column:

```python
with st.expander("How your data is used"):
    st.markdown(
        "**What we collect:** Your learning goal, selected persona, and optionally your "
        "resume text. During learning, we also record quiz scores and session timing.\n\n"
        "**AI-generated assessments:** Skill levels, learner profiles, and learning content "
        "are generated by AI based on the information you provide. They are estimates and "
        "may not fully reflect your actual abilities.\n\n"
        "**External services:** Your learning goal and background information are sent to "
        "an LLM provider (e.g., OpenAI) to generate personalised assessments and content.\n\n"
        "**Your control:** You can delete your account and all associated data at any time "
        "from the My Profile page. No data is shared with third parties or used for advertising."
    )
```

**Properties:**
- Collapsed by default (non-intrusive)
- No buttons, callbacks, or state changes
- Purely informational text
- Does not affect onboarding validation or navigation
